# Session context

Working notes for picking this project back up. Not part of the addon build.

## Goal

Replace the Node-RED "APK Update Service" flow with a self-contained Home
Assistant addon. The addon:

- Subscribes to a webhookrelay bucket over WebSocket.
- Receives GitHub `release` webhook payloads.
- Parses `versionCode` / `versionName` from the release body.
- Matches release assets to user-configured flavors.
- Downloads matching `assets[].url` (authenticated GitHub API URL, with
  redirect-followed S3 fetch) into `/data/apks/<filename>`.
- Serves `GET /api/latest.json?flavor=<id>` and `GET /api/download/<file>`
  for in-app Android self-updaters.
- Shows status in a small HA Ingress UI.

QR-code provisioning from the original Node-RED flow was intentionally
dropped.

## Configuration the user is filling in

In HA addon options:

- `webhookrelay.key` ŌĆö the Access Key from the webhookrelay dashboard
  Access Tokens page (UUID-looking string).
- `webhookrelay.secret` ŌĆö the **Main API key** at the top of the same
  page, starts with `sk-whrm-...`. The user's previous key was exposed
  in a screenshot; advised them to rotate it.
- `webhookrelay.bucket` ŌĆö bucket name or id that the GitHub webhook posts to.
- `github_token` ŌĆö PAT with repo:read; used as `Authorization: token ŌĆ”`
  against the asset API URL.
- `download_base_url` ŌĆö optional; leave blank to let the addon derive it
  from the incoming request. Set explicitly if Android clients hit the
  addon via a different hostname than the request URL would imply.
- `flavors` ŌĆö list of `{ id, filename_pattern }`. **Open design item,
  see below.**

## Open design item: replace regex `filename_pattern`

The user said they don't like the regex pattern. We started discussing
alternatives but didn't lock one in yet. The four options on the table
were:

1. **Include/exclude keyword lists** (recommended; clean for the
   `tablet` vs `tabletdpc` case):
   ```yaml
   - id: tablet
     contains: ["tablet"]
     not_contains: ["dpc"]
   - id: tabletdpc
     contains: ["tablet", "dpc"]
   - id: keypad
     contains: ["keypad"]
   ```
2. **Glob patterns** (e.g. `*keypad*.apk`) ŌĆö easy, but `*tablet*.apk`
   matches both tablet and tabletdpc, so ordering tricks are required.
3. **Gradle flavor name** ŌĆö parse `app-<flavor>-<buildType>.apk` and
   match the flavor segment exactly (`officeKeypad`, `basementTablet`, ...).
4. **Explicit filename** ŌĆö exact match per flavor.

Next time, pick one and update:

- `apk-update-service/config.yaml` (default options + schema).
- `apk-update-service/src/config.js` (zod schema + compile step).
- `apk-update-service/src/flavors.js` (matching function).
- `apk-update-service/src/routes/adminApi.js` (response shape passes the
  flavor config to the UI; update label).
- `apk-update-service/ui/app.js` (column header / display of the pattern).
- `apk-update-service/translations/en.yaml`.
- `apk-update-service/README.md` (options reference + examples).
- Bump version to `0.2.0` and add a CHANGELOG entry.

If the user picks option 1 (most likely), the matching function becomes:

```js
function matches(name, flavor) {
  const n = name.toLowerCase();
  const all = (flavor.contains || []).every((s) => n.includes(s.toLowerCase()));
  const none = (flavor.not_contains || []).every((s) => !n.includes(s.toLowerCase()));
  return all && none;
}
```

## Notable code details to remember

- `githubDownload.js` strips the `Authorization` header on the
  cross-origin redirect from `api.github.com` ŌåÆ S3, otherwise S3 rejects
  with a signed-URL conflict.
- `state.js` uses tmp-file-and-rename for atomic writes and serializes
  writes through a chained promise.
- The download route's filename guard is strict `^[A-Za-z0-9._-]+$` plus
  a resolved-path-prefix check against the APK dir.
- `webhookrelay.js` sends `{action:"auth",key,secret}` then
  `{action:"subscribe",buckets:[bucket]}`, pings every 25s, exponential
  backoff up to 60s.
- `latestJson.js` derives the base URL from `x-forwarded-proto` /
  `x-forwarded-host` when `download_base_url` is empty, and includes the
  `/api/download/` segment (the Node-RED original missed this).
